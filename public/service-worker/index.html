Service Worker

Background service that handles network requests. Ideal for dealing with offline situations and background syncs or push notifications.
Cannot directly interact with the DOM.

Example taken from Traversy Media

Steps

SW Cached Pages:

  • Test the "Without Service Worker":
    • Go to "Without Service Worker"
    • Refresh the page. It should work
    • Go offline (physically or through the dev tool)
    • Refresh the page. It should fail
  • Test the "With Service Worker":
    • Go to "With Service Worker"
    • Refresh the page. It should work
    • Go offline (physically or through the dev tool)
    • Refresh the page. It should work

SW Cached Site:

As long as you visit the page "With Service Worker", the Service Worker should start and therefore going offline and refresh will give you access to all the pages you already visited.
